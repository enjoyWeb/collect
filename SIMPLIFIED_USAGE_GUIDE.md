# 🚀 简化版使用指南

## 配置步骤（只需3步！）

### 第 1 步：输入 URL 列表

在**URL 列表**文本框中粘贴你的URL，**一行一个**：

```
https://www.amazon.com/gp/product/B01NBKTPTS
https://www.amazon.com/gp/product/B02QVZFG56
https://www.amazon.com/gp/product/B03XYZ1234
```

### 第 2 步：设置选择器

在**要点击的元素**输入框中，输入 CSS 选择器：

**已预设默认值：**
```css
.el-button.el-button--primary.el-button--mini
```

**你也可以输入其他选择器：**
- `.collection-btn` - 按class
- `#collect-button` - 按ID  
- `button[data-action="collect"]` - 按属性
- `a.btn:first-of-type` - 复杂选择器

### 第 3 步：应用配置

点击【✅ 应用配置】按钮

你会看到：
- URL 计数更新：`已添加：3 个URL`
- 日志显示：配置已应用

---

## 运行自动化

### 启动
1. 点击【▶️ 开始运行】
2. 扩展会依次打开每个URL
3. 自动点击指定的元素
4. 成功后自动关闭标签页
5. 打开下一个URL

### 暂停/继续
- 点击【⏸️ 暂停】暂停执行
- 点击【▶️ 继续】恢复执行

### 停止
- 点击【⏹️ 停止】停止所有操作

## 错误处理

### ❌ "请先【应用配置】"
- 原因：还没有点击【应用配置】
- 解决：输入URL和选择器后，点击【应用配置】

### ❌ "未找到选择器: .xxx"
- 原因：网页上找不到该选择器的元素
- 解决：
  1. 检查选择器是否正确
  2. 尝试在浏览器开发者工具中测试选择器
  3. 使用浏览器的数组工具找到正确的选择器

### ❌ "标签页响应错误"
- 原因：网页无法加载或脚本注入失败
- 解决：
  1. 确认URL是否可以正常打开
  2. 某些网站可能阻止脚本注入
  3. 重新加载扩展试试

## 常用选择器示例

### Amazon 采集按钮
```css
/* Element UI 风格的采集按钮 */
.el-button.el-button--primary.el-button--mini

/* 或者简化版 */
.el-button--primary
```

### 通用按钮选择器
```css
/* ID 选择 */
#collect-btn

/* Class 选择 */
.collect-button
.btn-collect
.action-collect

/* 属性选择 */
button[data-action="collect"]
a[onclick*="collect"]

/* 组合选择 */
div.container button.btn:first-of-type
```

### 查找正确的选择器

**方法 1：使用浏览器开发者工具**
1. 打开网页
2. 按 F12 打开开发者工具
3. 右键点击要点击的元素
4. 选择"检查"
5. 复制元素的 ID、Class 或其他属性作为选择器

**方法 2：使用 Console 测试**
1. 按 F12 打开 Console
2. 输入：`document.querySelector('.your-selector')`
3. 如果返回元素对象，说明选择器正确
4. 如果返回 null，说明选择器不存在

## ⚡ 快速技巧

### 技巧 1：批量复制URL
1. 在文本编辑器中整理好URL（一行一个）
2. 全选复制
3. 粘贴到扩展的URL输入框

### 技巧 2：使用预设值

- 保持日志区域可见
- 实时观察每个URL的处理状态

1. 只输入 1-2 个 URL 进行初始测试
2. 验证成功后再添加更多 URL
3. 遇到错误时更容易定位问题

## 📞 故障排除

### 问题：扩展无法点击元素

**怎么办：**
1. 打开网页
2. 按 F12 打开开发者工具
3. 在 Console 中输入：`document.querySelector('.el-button.el-button--primary')`
4. 检查是否返回元素
5. 如果返回 null，尝试其他选择器

### 问题：某些 URL 打开很慢

**怎么办：**
- 这是正常现象，扩展会自动等待 2 秒钟
- 如果网络特别慢，URL 可能超时
- 建议检查网络连接

### 问题：自动化中途停止

**怎么办：**
1. 检查日志中的错误信息
2. 查证该 URL 是否可以正常打开
3. 重新点击【开始运行】重试
