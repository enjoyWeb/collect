## 核心文件说明

### 1. manifest.json
**用途**: Chrome 扩展的配置文件（必需）
**关键信息**:
- 名称: 自动化采集工具
- 版本: 1.0.0
- Manifest 版本: 3（最新标准）
- 权限: tabs, storage, scripting, webNavigation
**何时修改**: 需要改变扩展名称、权限或添加新功能时

### 2. popup.html
**用途**: 扩展弹出窗口的 HTML 结构
**功能**:
- 配置区：输入 URL、选择器和延迟时间
- 链接列表：显示已添加的链接
- 控制区：开始、暂停、停止按钮
- 日志区：显示运行日志
**何时修改**: 需要改变 UI 布局时

### 3. popup.css
**用途**: 弹出窗口的样式表
**特性**:
- 响应式设计
- 美化的蓝紫色主题
- 优化的滚动条样式
- 按钮和表单的样式
**何时修改**: 需要改变外观或主题时

### 4. popup.js
**用途**: 弹出窗口的交互逻辑
**主要函数**:
- initConfig() - 初始化配置
- addUrl() - 添加 URL
- saveConfig() - 保存配置
- startAutomate() - 启动自动化
- renderUrlList() - 渲染列表显示
**何时修改**: 需要改变交互逻辑或添加新功能时

### 5. background.js
**用途**: 后台服务脚本（自动化的核心逻辑）
**主要功能**:
- 管理自动化状态
- 按序打开 URL
- 管理标签页生命周期
- 与 content script 通信
**关键函数**:
- openNextUrl() - 打开下一个链接
- handleAfterClick() - 处理点击后的逻辑
- notifyPopup() - 通知 UI 更新
**何时修改**: 需要改变自动化流程时

### 6. content-script.js
**用途**: 在网页中运行的脚本（页面注入）
**功能**:
- 监听来自后台脚本的消息
- 在页面中查找元素
- 模拟点击操作
- 滚动到元素位置
**关键函数**:
- clickElement() - 点击指定元素
- dispatchEvent() - 触发事件
**何时修改**: 需要改变点击行为或添加其他页面交互时

## 快速参考

### 常用快捷方式

| 操作 | 步骤 |
|-----|------|
| 打开设置 | 点击 Chrome 工具栏中的扩展图标 |
| 查看日志 | 在弹出窗口底部查看"运行日志" |
| 测试选择器 | "F12 → Console → document.querySelector('选择器')" |
| 查找元素属性 | F12 → 右键元素 → 检查 |
| 清除配置 | 扩展设置 → 清除浏览数据 |
| 检查扩展状态 | chrome://extensions → 查看此扩展 |

### 常用选择器

```css
.class-name           /* 按 class 查找 */
#element-id          /* 按 ID 查找 */
button               /* 按元素类型 */
[data-id="123"]      /* 按属性值 */
.class1.class2       /* 多个 class */
div > button         /* 直接子元素 */
div button           /* 后代元素 */
button:first-child   /* 第一个元素 */
```

## 常见问题快速答案

**Q: 如何找到正确的选择器？**
A: 打开 F12 → 右键元素 → 检查 → 复制相关属性

**Q: 运行时显示"未找到选择器"**
A: 检查选择器是否正确，或增加延迟时间

**Q: 扩展无法打开链接**
A: 确保 URL 格式正确（包含 http://），并且链接可访问

**Q: 自动化中断了怎么办？**
A: 查看运行日志，找到具体哪个链接失败，检查该链接的配置

**Q: 可以同时打开多个链接吗？**
A: 不行，扩展设计为按序处理

**Q: 支持 iframe 吗？**
A: 目前不支持，这是已知限制
