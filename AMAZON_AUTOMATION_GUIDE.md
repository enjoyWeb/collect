# Amazon 采集自动化指南 📦
## 使用扩展自动化"采集到趣天助手"流程

### 📌 场景说明

你的任务流程：
```
1. 打开 Amazon.co.jp 产品页面
2. 点击"★ 採集到趣天助手"按钮
3. 等待加载界面消失
4. 等待成功弹窗出现
5. 关闭弹窗
6. 重复下一个链接
```

这个新功能会自动帮你完成 **步骤 3-5**，你只需要准备 URL 列表！

---

## 🚀 快速配置步骤

### 第 1 步：准备 Amazon 链接

收集所有需要采集的 Amazon 产品 URL：

```
https://amazon.co.jp/dp/B0D4QVVJ3D
https://amazon.co.jp/dp/B0BZLV5T9P
https://amazon.co.jp/dp/B0BR8K4K1S
https://amazon.co.jp/dp/B0CPT8V7XY
...
```

### 第 2 步：打开扩展并粘贴链接

1. 在 Chrome 右上角点击扩展图标
2. 在"📋 输入 URL 列表"的 Textarea 中粘贴所有链接
3. 每个链接自动显示在下方

示意图：
```
┌─────────────────────────────────────┐
│  📋 输入 URL 列表                   │
│  https://amazon.co.jp/dp/B0D4Q...  │
│  https://amazon.co.jp/dp/B0BZL...  │
│  https://amazon.co.jp/dp/B0BR8...  │
└─────────────────────────────────────┘
                ↓
        ⚙️ 配置链接 (3)
        🔗 https://amazon.co.jp/dp/B0D4Q...
           ⚠️ 未配置选择器
           ☐ ⏳ 点击后等待"成功"弹窗

        [🎯 选择元素] [🗑️]
```

### 第 3 步：配置"采集到趣天助手"按钮

对于每个链接：

1. 点击 **【🎯 选择元素】** 按钮
2. 该 Amazon 页面会在新标签页打开
3. **鼠标悬停**在蓝色的"★ 採集到趣天助手"按钮上（会显示蓝色高亮）
4. **点击该按钮**
5. 扩展会自动显示提取的选择器

   ```
   示例：
   已选择: a.a-link-normal.a-color-base[data-a-target="collected-button"]
   ```

6. 点击 **【✅ 确认选择】** 按钮

### 第 4 步：启用"等待成功弹窗"

在配置好选择器后，你会看到每个 URL 项下方有一个复选框：

```
☐ ⏳ 点击后等待"成功"弹窗
```

**点击勾选这个复选框**，启用弹窗检测功能。

✅ 这样扩展会：
- 点击按钮
- 自动等待"成功"弹窗出现（最多 30 秒）
- 检测到成功后自动继续下一个链接
- 显示日志告诉你是否检测到成功

### 第 5 步：配置选项

在"🎮 运行控制"部分，确保：

```
✅ 页面弹出后自动关闭标签页      （勾选）
✅ 自动打开下一个链接             （勾选）
```

这样每处理完一个链接，页面会自动关闭。

### 第 6 步：开始自动化

点击 **【▶️ 开始运行】** 按钮

扩展会：
1. 打开第一个 URL
2. 点击"采集到趣天助手"按钮
3. ⏳ 等待成功弹窗（如果启用）
4. 自动关闭页面
5. 打开下一个 URL
6. 重复步骤 2-4
7. ✅ 所有完成时显示"完成"消息

## ⚙️ 高级选项

### 调整延迟时间

如果某个 Amazon 页面加载较慢，需要更长的等待时间，你可以：

```
当前：等待 2 秒后点击
可改为：等待 3-5 秒（取决于网络速度）
```

（可在后续版本中添加 UI 调整功能）

### 弹窗检测说明

扩展会自动搜索页面中包含以下内容的元素：
- 包含"成功"字样的弹窗
- 常见的弹窗类名：`.popup`, `.modal`, `.alert`, `.message` 等
- 所有可见的文本元素

最多等待 **30 秒**，然后自动继续。

## 🐛 常见问题排查

### Q1: 按钮选择器无法提取？

**症状：** 点击按钮后显示 `-`（未识别）

**原因：** 
- 页面加载不完全
- 点击的不是实际按钮

**解决：**
1. 等待页面完全加载后再点击
2. 确保点击的是"采集到趣天助手"按钮（应该是蓝色的）
3. 试试点击按钮的文字部分

### Q2: 自动化运行但没有点击？

**症状：** 看到日志"已点击元素"但页面没反应

**原因：**
- 选择器不准确
- 页面元素可能动态变化

**解决：**
1. 重新选择一遍元素
2. 增加延迟时间
3. 检查浏览器开发者工具的控制台看是否有错误

### Q3: 一直检测不到成功弹窗？

**症状：** 日志显示"未找到成功弹窗"

**原因：**
- 弹窗中文显示可能不是"成功"
- 刷新后弹窗消失了
- 弹窗可能是其他形式（Toast、对话框等）

**解决：**
1. 手动点击试试，看弹窗具体样子
2. 如果确实有成功提示，禁用"等待弹窗"，直接使用延迟时间
3. 增加延迟时间到 3-5 秒

### Q4: 错误：找不到选择器？

**症状：** 日志显示"❌ 未找到选择器"

**解决：**
1. 确保选择器正确（点击"选择元素"重新提取）
2. 检查是否勾选了错误的 URL 项
3. 试试点击"选择元素"使用新的选择器
