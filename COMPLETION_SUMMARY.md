## 🎯 核心功能说明

### 1. URL 输入和配置

```
┌─────────────────────────────────────┐
│  输入 URL 列表（Textarea）          │
│  https://example.com/page1          │
│  https://example.com/page2          │
│  https://example.com/page3          │
└─────────────────────────────────────┘
                  ↓
     系统自动生成 URL 列表
       ✅ 已配置 / ⚠️ 未配置
         [🎯 选择元素] [🗑️ 删除]
```

### 2. 元素选择工作流

```
用户点击"🎯 选择元素"
     ↓
页面在新标签页打开
     ↓
启用元素选择模式
     ↓
用户鼠标悬停元素（显示蓝色高亮）
     ↓
用户点击目标元素
     ↓
系统提取 CSS 选择器并显示
     ↓
用户点击"✅ 确认选择"保存
     ↓
页面自动关闭，返回扩展 UI
```

### 3. 自动化执行流程

```
点击"▶️ 开始运行"
     ↓
遍历每个 URL：
  1. 打开 URL
  2. 等待延迟时间（2秒）
  3. 向页面注入 content-script
  4. 执行点击操作
  5. 根据设置选择：
     - 关闭页面并继续
     - 或保留页面手动处理
     ↓
所有 URL 处理完成
     ↓
显示完成消息和日志
```

## 🔐 权限说明

manifest.json 中声明的权限：

| 权限 | 用途 |
|-----|------|
| `tabs` | 打开、关闭、查询标签页 |
| `storage` | 保存和读取配置 |
| `webNavigation` | 监听页面导航事件 |
| `scripting` | 注入脚本到页面 |
| `activeTab` | 与当前活跃标签页交互 |

所有权限都是实现功能所必需的。

## 🐛 已知限制

1. **iframe 元素**
   - 暂不支持选择 iframe 内的元素
   - 可能的改进：注入 iframe 内的脚本

2. **Shadow DOM**
   - 暂不支持 Shadow DOM 中的元素
   - 原因：Shadow DOM 的隔离特性

3. **动态生成的内容**
   - 对于 React/Vue 等框架，需要等待内容完全加载
   - 解决方案：手动调整延迟时间

4. **跨域限制**
   - Content Script 只能在清单中声明的源上运行
   - 跨域请求可能被限制
