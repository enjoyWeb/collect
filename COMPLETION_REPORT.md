## 🔄 功能对比

### 旧版本（复杂模式）
```
输入URL → 逐个选择元素 → 为每个URL配置选择器 → 运行自动化
成本：高（需要逐个配置）
代码：复杂（各种per-URL处理）
```

### 新版本（简洁模式）
```
输入URL + 输入选择器 → 应用配置 → 运行自动化
成本：低（一次性配置）
代码：简洁（统一处理）
```

## 🚀 使用流程

### 配置流程（新）
```
1. 打开扩展
2. 输入URL列表（多行复制粘贴）
3. 输入CSS选择器（或使用默认值）
4. 点击【应用配置】
5. 点击【开始运行】
✓ 完成！平均用时：< 2 分钟
```

### 旧流程对比
```
1. 打开扩展
2. 输入第1个URL
3. 点击【选择元素】
4. 点击目标元素
5. 确认选择器
6. 对每个URL重复2-5步...
7. 点击【开始运行】
✗ 平均用时：> 10 分钟
```

## 🎓 技术详情

### 配置格式变化
```javascript
// 旧格式
{
  urls: [
    { url: "...", selector: null, delay: 2000, waitForPopup: false },
    { url: "...", selector: null, delay: 2000, waitForPopup: false },
  ]
}

// 新格式
{
  urls: ["...", "..."],
  selector: "...",
  running: false,
  paused: false,
  currentIndex: 0
}
```

### 数据流简化
```javascript
// 旧：复杂的per-URL处理
urlItem.url
urlItem.selector
urlItem.delay
urlItem.waitForPopup

// 新：统一处理
url                    (直接字符串)
config.selector        (统一选择器)
固定延迟2秒          (统一值)
始终自动打开下一个    (固定行为)
```
